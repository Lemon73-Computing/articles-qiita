## TL;DR

1. GitLab でレポジトリの Https クローン URL をコピー
1. GitHub Desktop で URL からローカルレポジトリを作成
1. (コミットすると GitLab に反映される)

## こちらの文章について

GitHub Desktop を使い、GitLab にコミットする方法が思ったより簡単だったので、それを紹介する記事です。

## 前提条件

- GitHub アカウント所持
- GitLab アカウント所持
- GitLab にレポジトリがある

## 方法

### 1. クローンで用いる URL を取得

(GitLabでの作業)

まず初めに、ローカルで作業したいレポジトリの Https クローンの URL を取得します。

![1.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/2769460/15ea8159-013d-78f2-a009-20cde9aaabdc.png)

### 2. GitHub Desktop でクローン

(GitHub Desktopでの作業)

続いて、これを GitHub Desktop にてクローンします。

上側のリボンの **File** を選択して、上から3番目の **Clone a repository** を選択すると下の画面が出てきます。

![2.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/2769460/c10f6982-3669-bf55-6294-900ec90a3185.png)

出てきたら上側に先ほどコピーしたURLを、下側はローカルのお好きな場所を選択してください。
これで完了です。

### 3. コミットしてみる

では、実際にコミットしてみましょう。

![3.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/2769460/7965845e-e690-2322-f8f7-24cefdac7ad9.png)

今回は **readme** という名前でコミットしてみます。

(はじめてコミットするときは GitLab のユーザー名とパスワードを求められます)

![4.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/2769460/3c39e494-1cc2-d34e-0c05-a2712113d4c6.png)

GitLab 側で確認すると、しっかり **readme** という名前でコミットできており、成功したのがわかります。

## まとめ

GitHub Desktop を利用することで、コマンド操作を減らすことができるため、初学者や非プログラマーが Git を使いやすい環境を作ることができると考えています。

本記事の手順を踏むことで、GitHub だけでなく GitLab でも利用できます。皆さんもぜひお試しください。
